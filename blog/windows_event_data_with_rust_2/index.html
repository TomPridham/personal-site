getting windows event data with rust part 2

back in part 1 we successfully subscribed to the logout/login windows events and were able to print them to the console. but that's only minimally useful if we can't send that information back to the rest of our program somehow. one way to get that data back into our main program is via mpsc channels. mpsc stands for `multiple producer, single consumer`. which means that there will be a single handler, but you could potentially subscribe to multiple different windows events with the same channel.

first we need to make our sender available to be used within our callback. we ended the last part with this.


let session = 0;
let signal_event = None;
let channel_path = "Security";
let query = "EventID=4624 or EventID=4634";
let bookmark = 0;
let context = std::ptr::null_mut();
let flags: u32 = EventLog::EvtSubscribeToFutureEvents.0 as u32;
unsafe {
    EventLog::EvtSubscribe(
        session,
        signal_event,
        channel_path,
        query
        bookmark,
        context,
        Some(event_callback),
    );
}

instantiating the channel is pretty easy. surely it won't be this easy to use it?


//snip
let (sender, mut receiver) = unbounded_channel();
let context = sender;
unsafe {
    EventLog::EvtSubscribe(
        //snip
        context,
        Some(event_callback),
    );
}

loop {
    if let Some(s_event) = receiver.recv().await {
        println!("received event: {}", s_event.trim());
    }
}

nope. trying to run this we get the following error: `expected enum 'c_void', found struct`. so now we need to turn that struct into a raw pointer to be able to pass it as our context. to do this we need to familiarize ourselves with some of the spookier(imo) parts of unsafe rust, dealing with raw pointers. we can turn a reference to our struct into a compatible c pointer in safe rust. the docs for c_void talk about it better than i could and link to more information on pointers. but the gist of what we are doing is coercing a reference to our producer into a const pointer, then casting it as c_void pointer.


//snip
let (mut sender, mut receiver) = unbounded_channel();
let context = &mut sender as *mut UnboundedSender<String> as *const c_void;

unsafe {
    EventLog::EvtSubscribe(
        //snip
        context,
        Some(event_callback),
    );
}

loop {
    if let Some(s_event) = receiver.recv().await {
        println!("received event: {}", s_event.trim());
    }
}

now inside of our callback we need to turn that pointer back into a usable rust struct. this does require unsafe rust because the normal guarantees that might prevent a dangling pointer can't be upheld by the compiler once the pointer has passed the ffi boundary. first we need to convert back into a pointer of the correct type, then we need to dereference that pointer so we can access the actual struct.


#[no_mangle]
extern "system" fn event_callback(
    action: EventLog::EVT_SUBSCRIBE_NOTIFY_ACTION,
    p_context: *const c_void,
    h_event: isize,
) -> u32 {
    //snip
    //
    let sender_ptr = p_context as *mut UnboundedSender<String>;
    let sender = unsafe { &mut *sender_ptr };
    //snip
}

there is still a problem here though. the sender will get dropped after the first invocation of the callback, which isn't great because that subscription is relying on the callback function working until it is unsubscribed(i.e. forever). so after the first invocation of the callback we will have an invalid pointer. we need to do something to make the sender persist for as long as the program does. enter Box::leak. this will let us put the sender on the heap and then leak it to prevent it's destructor from being run.


let (mut sender, mut receiver) = unbounded_channel();

let sender = Box::new(sender);
let s = Box::leak(sender.clone());

let context = s as *mut UnboundedSender<String> as *const c_void;
unsafe {
    EventLog::EvtSubscribe(
        //snip
        context,
        Some(event_callback),
    );
}

the last thing we need to do is update our pointer conversion in the callback.


#[no_mangle]
extern "system" fn event_callback(
    action: EventLog::EVT_SUBSCRIBE_NOTIFY_ACTION,
    p_context: *const c_void,
    h_event: isize,
) -> u32 {
    //snip
    //
    let sender = unsafe {
        (p_context as *mut UnboundedSender<String>)
            .as_ref()
            .unwrap()
    };
    //snip
}
